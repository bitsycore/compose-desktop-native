# JvmDefaultWithCompatibility per-package nonJvm actuals.
#
# What it is: a Kotlin annotation that on JVM controls how interface default
# methods compile to bytecode (older "DefaultImpls" companion class) for
# binary-compatibility. On nonJvm (Native) it's typealiased to a `@NoOp`
# annotation — does literally nothing at runtime.
#
# Why we vendor it: purely an import-resolution requirement. Vendored
# upstream files reference `@JvmDefaultWithCompatibility` on their
# interfaces; the import must resolve. Per-module entries are added ONLY
# when a vendored file in that module actually imports the annotation.
